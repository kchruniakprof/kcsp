## 4. Najważniejsze, czego nie zgubić (różnice vs DKV)

Pełny opis: [`plan.md` §2, §9](./plan.md). Skrót:
1. **2 niekompatybilne schematy numeracji** (Kfz litery A–N, Hausrat numery 1–30) → parser generyczny hierarchii, nie regex jak DKV.
2. **Near-duplikaty Hausrat** (~90% wspólne Smart/Best/+moduły) → osobne pliki + filtr tarifu; COMPARE = runtime diff-step.
3. **Produkt 3-poziomowy, 4 Sparten** (Sparte>Tarif>Baustein), nie płaski jak DKV.
4. **OCR-ligatury docling** (`Haftpfl icht`, `fi nden`) → sanityzacja MUSI scalać fi/fl **przed** parserem.
5. **Cross-lingual** (query PL/EN, docs DE) → query expansion normalizuje terminy do DE.
